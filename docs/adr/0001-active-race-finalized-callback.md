# ActiveRace signals finalization via an injected callback, not a return value

`ActiveRace.finalize()` does not return a result — it fires an `onFinalized` callback injected at construction. The reason is the auto-finalization timer: when a lane crosses the target distance during `tick()`, `ActiveRace` starts a 1500ms grace-period timer so the trailing lane can contribute one last sample. When that timer fires, it must trigger the same post-finalization side effects (tournament outcome, result presentation, auto-stage, notifications) as admin-triggered finalization — but a timer callback cannot return a value to a caller that finished long ago. An injected callback handles both paths uniformly. Returning `FinalizedRaceResult` from `finalize()` was rejected because it would require the app to wire a separate timer callback anyway, splitting the finalization logic across two code paths.
